 *****Key Features*****

Three-role authentication — Student, Coordinator, Rector.

Smart approval workflow:

Working day → Coordinator + Rector approval

Non-working day → Only Coordinator approval

Digital leave request system with date, type, and reason.

Student dashboard showing complete leave history and live status.

Coordinator panel for verifying and approving/rejecting leave requests.

Rector panel for final approval of working-day leaves.

LocalStorage-based data persistence (no backend required for demo).

Fully responsive UI built with HTML, CSS, Bootstrap-style layout.

React version available for modern component-based usage.
